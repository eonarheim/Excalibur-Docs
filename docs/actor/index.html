Actor

The most important primitive in Excalibur is an "Actor." Anything that can move on the screen, collide with another Actor, respond to events, or interact with the current scene, must be an actor.

Usage

var game = new ex.Engine();

// Create new actor at x = 50, y = 50, width = 10, height = 10, and color = black
var actor = new ex.Actor(50, 50, 10, 10, Color.fromHex('#000000'));

game.addChild(actor);

game.start();

Constructor

new(x? : number,  
   y? : number,  
   width? : number, 
   height? : number, 
   color? : Color)

The Actor constructor takes 5 optional parameters: x position, y position, width, height, and color. If no parameters are specified, the default values of x, y, width, and height will all be 0. Color will default to black.

Static Methods

actor.extend(methods : any)

You may wish to extend native actor functionality. Any method on actor may be extended to support additional functionaliy. In the example below we create a new type called "MyActor"

var MyActor = ex.Actor.extend({
   update : function(engine, delta){
      // Implement custom update 

      // Call super constructor update
      this.super.update.call(this, engine, delta);
      console.log("Something cool!");
   }
});

var myActor = new ex.MyActor(100, 100, 100, 100, Color.Azure);

Note - If you are using TypeScript, it is easier to use the built-in inheritance mechanic . This method is meant as a convenience for games built in JavaScript.

Properties

actor.x

The x position of the actor in game coordinates. Defaults to 0.

actor.y

The y position of the actor in game coordinates. Defaults to 0.

actor.rotation

The current rotation of an actor in radians. Defaults to 0.

actor.scale

The current scale of an actor. Defaults to 1.

actor.dx

The current x velocity of an actor. Defaults to 0.

actor.dy

The current y velocity of an actor. Defaults to 0.

actor.rx

The current angular velocity of an actor. Defaults to 0.

actor.sx

The current scale velocity of an actor. Defaults to 0.

actor.invisible

Get or set the current visibility of an actor. If 'invisible' is set to true, then the actor will no longer be drawn, but will still receive updates.

actor.fixed

If 'fixed' is set to true the actor will not be pushed by other actors on collision. If 'fixed' is set to false then other actors will influence the current actor's position on collision.

actor.frames

Dictionary containing all of the drawable images/animations for an actor. You should not be adding to this directory. Use the 'addDrawing' method below.

actor.currentDrawing

The current drawing being displayed for this actor. You should not ever set this property. Use the 'setDrawing' method below.

actor.color

The current color for an actor. This is only used if no drawings have been specified for an actor.

actor.parent

The parent property represents the SceneNode that is this actor's parent. This property is null by default and is set when the actor is added to the scene graph.

Methods

actor.kill()

This is a convenience method to remove the current actor from the scene. Actors that are removed from the scene will no longer be drawn or updated. 

actor.addChild(actor: ex.Actor)

Adds a child actor to this actor. This allows you to move the parent actor around and have children automatically follow.

Note All child coordinates are relative to the parent.

actor.removeChild(actor: ex.Actor)

Removes a child actor from this actor.

actor.addDrawing(key: string, drawing: ex.IDrawable)

Adds a drawing to the collection of available drawings for an actor.

actor.setDrawing(key : string)

Set the current drawing for the actor from the available drawing collection.

actor.setCenterDrawing(isDrawingCentered: boolean)

Centers the current drawing around the actor's bounding box.

actor.addEventListener(eventName: string,  
   handler: (event?: ex.ActorEvent) => void)

Add an event listener to this actor. You can listen for a variety of events off of the actor; see the events section below for a complete list.

actor.triggerEvent(eventName: string, event?: ex.ActorEvent)

Trigger an event to occur off of an actor.

actor.getCenter()

Returns a vector with the x and y position of the center of the actor.

actor.getWidth()

Returns the width of an actor, factoring in the scale.

actor.setWidth(width: number)

Sets the width of an actor, factoring in the scale.

actor.getHeight()

Returns the height of an actor, factoring in the scale.

actor.setHeight(height: number)

Sets the height of an actor, factoring in the scale.

actor.getLeft()

Returns the x position of the left side of the actor's bounding box.

actor.getRight()

Returns the x position of the right side of the actor's bounding box.

actor.getTop()

Returns the y position of the top side of the actor's bounding box.

actor.getBottom()

Returns the y position of the bottom side of the actor's bounding box.

actor.contains(x: number, y: number)

Returns true if the x and y position specified are inside or on the edge of the actor's bounding box.

actor.collides(other: ex.Actor)

Returns the 'Side' of the other actor if actor collides with the other actor. Otherwise it returns the 'None' side.

actor.within(other: ex.Actor, distance: number)

Returns true if the two actors are less than or equal to the distance specified from each other.

actor.moveTo(x: number, y: number, speed: number

This method will move an actor to the specified x and y position at the speed specified (in pixels per second) and return back the actor.

This method is part of the actor 'Action' fluent API allowing action chaining. 

actor.moveBy(x: number, y: number, time: number

This method will move an actor to the specified x and y position by a certain time (in milliseconds).

This method is part of the actor 'Action' fluent API allowing action chaining. 

actor.rotateTo(angleRadians: number, speed: number

This method will rotate an actor to the specified angle at the speed specified (in radians per second) and return back the actor.

This method is part of the actor 'Action' fluent API allowing action chaining. 

actor.rotateBy(angleRadians: number, time: number

This method will rotate an actor to the specified angle by a certain time (in milliseconds) and return back the actor.

This method is part of the actor 'Action' fluent API allowing action chaining. 

actor.scaleTo(size: number, speed: number

This method will scale an actor to the specified size at the speed specified (in magnitude increase per second) and return back the actor.

This method is part of the actor 'Action' fluent API allowing action chaining. 

actor.scaleBy(size: number, time: number

This method will scale an actor to the specified size by a certain time (in milliseconds) and return back the actor.

This method is part of the actor 'Action' fluent API allowing action chaining. 

actor.blink(frequency: number, duration: number, blinkTime? number)

This method will cause an actor to blink (become visible and and invisible) at a frequency (blinks per second) for a duration (in milliseconds). Optionally, you may specify blinkTime, which indicates the amount of time the actor is invisible during each blink.

To have the actor blink 3 times in 1 second, call actor.blink(3, 1000).

This method is part of the actor 'Action' fluent API allowing action chaining. 

actor.delay(time: number)

This method will delay the next action from executing for a certain amount of time (in milliseconds).

This method is part of the actor 'Action' fluent API allowing action chaining. 

actor.repeat(times?: number)

This method will cause the actor to repeat all of the previously called actions a certain number of times. If the number of repeats is not specified it will repeat forever.

This method is part of the actor 'Action' fluent API allowing action chaining. 

actor.repeatForever()

This method will cause the actor to repeat all of the previously called actions forever.

This method is part of the actor 'Action' fluent API allowing action chaining. 

actor.update(engine: ex.Engine, delta: number)

This method is called by the engine to update each actor. Delta is the amount of time elapsed since the last time update was called (in milliseconds).

You should only be calling this method if you wish to extend actor.

actor.draw(ctx: CanvasRenderingContext2D, delta: number)

This method is called by the engine to draw each actor. Delta is the amount of time elapsed since the last time draw was called (in milliseconds).

You should only be calling this method if you wish to extend actor.

actor.debugDraw(ctx: CanvasRenderingContext2D)

This method is called by the engine when in debug mode. This will add general debug information to the screen.

Events

Valid events to listen for off of Actor.

'keydown'
'keyup'
'keypressed'
'update'
'click'
'mousedown'
'mouseup'

See the events documentation for specifics about each of these events.
